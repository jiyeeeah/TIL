#### 락 (Lock)

- 정의
  - 하나의 스레드나 프로세스가 자원을 점유한 경우, 다른 스레드나 프로세스가 접근하지 못하도록 제한하는 것
- 비고
  - 공유 자원의 동시성 문제를 해결하기 위해 사용된다.
  - 원자성 보장을 통한 데이터 무결성을 보장하기 위해 사용된다.
    - 락을 통해서 공유 자원에 단 하나의 스레드나 프로세스만 접근을 허용함으로써, 해당 스레드나 프로세스가 작업이 끝날 때까지 다른 스레드나 프로세스는 접근하지 못하게 된다. 이로 인해 원자성이 보장되고, 데이터 무결성이 보장되게 된다.
  - 락을 무분별하게 사용할 경우 성능이 저하될 수 있다.
- 동작 방식
  - 공유 자원은 대기 상태
    → Thread 1이 공유 자원에 접근하기 위해 락을 요청
    → Thread 1은 성공적으로 공유 자원에 대한 락을 점유
    → 이 상태에서 Thread 2가 공유 자원에 락을 요청하면 이미 Thread 1이 점유하고 있기 때문에 Thread 2는 해당 요청을 보류하고 대기 상태에 들어가게 된다.

#### 원자성

- 정의
  - 처음부터 끝까지 실행되거나 실행되지 않은 상태
- 비고
  - 원자성이 깨졌다? = 실행이 중간에 끊겼다.
  - 예를 들어 `count++;` 의 경우 `count`라는 `값 읽기`, `증가시키기`, `쓰기`의 세가지 작업이 실행되어야 하는데, `증가시키기`까지 진행되고 컨텍스트 스위칭이 일어나버리면 원자성이 깨진 경우라고 할 수 있다.

#### 뮤텍스 (Mutex)

- 정의
  - Mutual Exclusion의 약어로, 상호 배제를 의미한다.
  - 하나의 프로세스/스레드가 임계영역에서 실행될 때 다른 프로세스/스레드는 임계영역에 접근할 수 없도록 하는 것
- 비고
  - 뮤텍스는 조건 변수와 함께 사용하게 된다.
    - 조건 변수를 통해 특정 조건이 충족되면 signal을 보내 대기 중인 스레드를 깨움
  - **락의 소유권과 해제 권한을 특정 스레드에 부여하는 것이 특징**
  - 한 번에 하나의 스레드만 접근할 수 있도록 하기 때문에 데이터 무결성을 보장
  - 락이 어떤 스레드에 의해 점유되어 있으면, 락 요청시 해당 스레드를 **뮤텍스 대기 큐**에 넣음
    락이 해제되면 뮤텍스 대기 큐의 첫번째 스레드에 notify → 해당 스레드는 락 요청 → 락 획득
    FIFO로 접근하게 됨

#### 세마포어 Semaphore

- 정의
  - 공유 자원에 동시 접근할 수 있는 허용량을 제한하는 것
- 비고
  - 상호배제와 유한한 대기를 충족시키기 위해 사용된다.
    - 세마포어 값을 획득해야 공유자원에 접근할 수 있음 (세마포어 값 = 남는 자리가 있는지)
  - 다중 접근 제한

#### 뮤텍스와 세마포어

- 뮤텍스는 사용 중인 프로그램이 끝나기 전에 다른 프로그램이 끼어들 수 없기 때문에 **파일 I/O**나 **데이터 베이스 갱신**에서 사용하는 것이 적절
  - 파일 I/O 과정에서 다른 프로그램이 끼어들게 되면 파일이 온전하게 전달될 수 없음
  - 데이터 베이스를 갱신할 때도 중간에 다른 프로그램이 끼어들면 값이 온전하지 못할 수 있음
- 세마포어는 공유 자원에 대한 접근 허용량을 지정해 주기 때문에, **네트워크 접속 제한**이나 **스레드 풀 관리**에서 사용
  - 서버의 리소스는 한정되어 있기 때문에 네트워크 접속량을 제한함으로 공유 자원을 보호할 수 있음
  - 그리고 동시에 실행될 수 있는 스레드를 제한함으로 시스템 리소스를 효율적으로 관리할 수 있음
